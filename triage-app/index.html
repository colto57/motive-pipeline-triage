How we score

Every score is computed in-browser from your CSV fields — no opaque model. We calibrate against n=38 active Motive venture investments on motivepartners.com/portfolio (41 Venture entries on site; 3 realized exits excluded from the similarity corpus: Corastone, Februar, Vitera). Hard filters run first; in-mandate companies get a weighted sum of 14 component scores, a spread transform, and a small tiebreaker.

Scoring formula

  1. Hard filters — stage and geography only. Fail either → filtered out (score 0).
  2. Component scores — each factor returns 0–100 using the rules below.
  3. Weighted raw — Σ (component × weight) across 14 factors (weights sum to 100%) plus a tiebreaker bonus (0–2 pts).
  4. Spread transform — stretches mid-pack scores so tiers separate cleanly: raw ≥74 → 74 + (raw−74)×1.4; ≥62 → 62 + (raw−62)×1.1; ≥48 → 48 + (raw−48)×0.95; else 48 − (48−raw)×0.8.
  5. Overall score — spread result rounded to one decimal, clamped 0–100.
  6. Tier — ≥76 Priority Review · 60–75 Standard Review · <60 Low Priority.

Tiebreaker (+0.2–2.0): disclosed ARR tiers, growth ≥50%, design partners, hub city, portfolio TF-IDF match ≥0.08, serial founder, tier-1 operator background. Used only when overall scores tie; ranking then falls back to traction → founder execution → velocity → thesis → raise size → name.

Hard filters (mandate gates)

Only two gates remove a company from the shortlist. Sector is not hard-filtered.

Stage gate
stage must normalize to pre-seed, seed, or series a. Series B+ → filtered with routing note.
Geography gate
hq_geography must parse as US (state abbreviation, “United States”, “US/USA”) or Europe (UK, Germany, France, Netherlands, Belgium, Spain, etc. — full list in code). Outside US/Europe → filtered.

Soft sector deprioritization

Adjacent tech sectors stay in the shortlist but score lower on portfolio sector fit (cap 5–42 vs core fintech 50–100).

Adjacent patterns
HR Tech, PropTech/CRE ops, logistics/supply chain, pure healthcare IT, standalone climate/ESG, non-wealth real estate tech — matched via regex on sector + pitch_summary + founder_background.
Penalty rules
Partial fintech overlap: round(50 + portfolioWeight×140) − 22, capped 5–42. No overlap: 18. Unclassified weak fintech signal: 28.

Scoring components (14 factors)

Thesis similarity16%
What it measures
How closely the inbound pitch language matches Motive’s venture thesis and portfolio corpus.
How it's calculated
Tokenize company_name, sector, founder_background, pitch_summary, stage, hq_geography, founding_year. Build TF-IDF vectors; cosine vs composite of 7 thesis statements + 38 portfolio company docs. Display score: round(25 + cosine × 400), clamped 0–95. Raw cosine typically 0.02–0.18.
Example
Embedded payments API + open banking language → cosine ~0.14 → 81. Generic HR workforce SaaS with no fintech terms → cosine ~0.04 → 41.
Data source
All CSV text fields above; reference corpus from MOTIVE_THESIS_STATEMENTS + MOTIVE_VENTURE_PORTFOLIO.
Portfolio sector fit17%
What it measures
Alignment with Motive’s observed venture sector mix (Wealth 34%, Banking/Payments 32%, AI/Data 16%, Capital Markets 11%, Insurance 5%, Business Services 3%).
How it's calculated
Regex-match sector + pitch_summary + founder_background against MOTIVE_SECTOR_TAXONOMY. Core fintech: round(50 + primary.portfolioWeight × 140); +8 if multi-vertical match; +6 if AI/agent/LLM/automation in sector or pitch. Adjacent: see soft deprioritization above.
Example
Wealth/advisor workflow (34% weight) → 50 + 0.34×140 ≈ 98. PropTech primary with partial payments overlap → capped near 38.
Data source
sector, pitch_summary, founder_background.
Traction8%
What it measures
Revenue, volume, growth, and institutional validation parsed from the pitch.
How it's calculated
Base 28. From pitch_summary: ARR tiers — ≥$10M +48, ≥$5M +38, ≥$1M +28, ≥$500K +18, else +6. GMV ≥$100M +32; ≥$25M +18. Growth ≥100% YoY/QoQ +20; ≥50% +12. Customer count phrase +10. Design partner / tier-1 bank pilot +14. Pre-revenue without pilots: −18. Clamped 0–100.
Example
“$2.1M ARR, 120% YoY, 3 tier-1 bank design partners” → ~28+28+20+14 = 90. “Pre-revenue, no pilots” → ~10.
Data source
pitch_summary (regex for $X ARR, $X GMV, N% YoY/QoQ, customer counts, design partners).

Company age fit6%
What it measures
Whether company age at submission matches typical Motive entry timing for that stage.
How it's calculated
Age = 2026 − founding_year. Ideal windows: Pre-Seed 0–2 yr (acceptable to 3), Seed 1–4 (to 6), Series A 2–5 (to 7). In ideal window: 95 minus distance from center (up to −12). Young for stage: 68–74. Slightly over: 78 − 6×years over. Mature: max(45, 58 − 5×years over). Missing year → 50.
Example
Seed founded 2023 (3 yrs) → 91. Series A founded 2017 (9 yrs) → 48.
Data source
founding_year, stage.

Example
Seed (42% of corpus) → 95. Pre-Seed (21%) → 76.

Vertical AI fit5%
What it measures
AI applied to specific financial workflows — Motive’s 2024–2026 investment theme.
How it's calculated
AI signal (AI/agent/LLM/ML/automation) + financial workflow (AP/AR, underwriting, treasury, claims, etc.). Both → 92. AI only → 52. Workflow only → 68. Neither → 42. Generic “AI-powered everything” −15. Regulatory tailwind (PSD2, embedded finance) +8.
Example
“LLM agents for loan underwriting compliance” → 92. “General-purpose chatbot” → 37.
Data source
sector, pitch_summary.
Traction velocity index5%
What it measures
Monthly ARR velocity (ARR ÷ months since founding) vs stage benchmarks from portfolio analytics.
How it's calculated
Monthly velocity = ARR / max(6, age in months). Compare to p25/p50/p75 by stage: Pre-Seed ~$2.5K/$9K/$22K per month; Seed ~$14K/$28K/$55K; Series A ~$45K/$100K/$200K. At/above p75 → 92; p50 → 78; p25 → 62; below → 45. Growth ≥100% +12; ≥50% +7. No ARR → 38–58 fallback.
Example
$1.2M ARR, 2 yrs Seed → ~$50K/mo (above p50) → 78. $200K ARR, 3 yrs Series A → 45.
Data source
pitch_summary, founding_year, stage.
